Overview

LoadPreset loads all the presets (both built-in and user-created) into a List<MediaFormat> and returns it.
The built-in presets are hard-coded into this function.
The user-created presets will be retrieved with the ReadMediaFromFile (Core.cs - Core) function, it is called within this function.

Definition

public static List<MediaFormat> loadPresets(bool console)

The function takes a bool to indicate if it is run by the console version. This is only used for the Output (Core.cs - Core) function.
The return is a MediaFormat list, that contains all the relevant disk information for FAT, as well as information for the preset (like the preset name)

Behavior

The function creates a local variable (Presets) with the List<MediaFormat> Type. This saves the return value.
After that, the function adds the built-in presets to the Presets variable.

Then, inside a try block, The Function goes through every file in the preset directory and uses ReadMediaFromFile (Core.cs - Core) to read the media format.
It will then be checked if it is a valid return. Then we add the file path of the preset to the media format and add it to the presets variable.
If an exception occurs, then Output (Core.cs - Core) is called with the unable_load_preset text.

After that, the Presets variable gets returned.
