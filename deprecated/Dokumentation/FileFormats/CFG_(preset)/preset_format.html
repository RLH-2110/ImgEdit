Overview

The preset CFG file saves Presets for the Image format, like Sides, Heads, mediaDescriptor, ...
It also saves meta information about the Preset, like the preset name.

Format

Format Version 1.0

imporant:
The first byte is refered to as byte 0
Byte adressed and values in this dokumentation are written in Hexadecimal!
All data is saved in little endian!
The progamm checks if the preset file is at least 0xB bytes long, the preset file can not be smaller than 0xB bytes!

 Variable
Version number
The first two bytes are the version. the first byte is the mayor version, the second byte is the minor version.
higer minor versions can be read by lower minor versions, but higher mayor versions can not be read my lower mayor versions.

Bytes per sector
byte 0x2 and 0x3 are to store the bytesPerSector of the media

Tracks per side
byte 0x4 and 0x5 are to store the tracksPerSide of the media

Sectors per track
byte 0x6 stores the sectorsPerTrack of the media

Sides
byte 0x7 stores the amount of sides of the media

Media descriptor
byte 0x8 stores the mediaDescriptor of the media

Preset name length
byte 0x9 stores the length of the preset name, it must be a value between 1 and FF

Preset name
The Preset name starts at byte 0xA and its length is determined by byte 0x9 (Preset name length). It has a minimum size of 0x1, and a maximum size of 0xFF


Data after the Preset name will be ignroed.
