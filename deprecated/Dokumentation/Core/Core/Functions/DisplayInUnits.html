Overview

This function converts bytes into the best unit (Bytes, KiB, MiB, GiB), it can also be configured to output values in (Bytes, KB,MB,GB)

Definition

public static DisplayInUnits_return DisplayInUnits(decimal value, bool kilo)
This function takes in a decimal, which is the number that we want to get the corret unit for. (I belive its a decimal, because NumericUpAndDown uses decimals)
the bool indicates if we use the 1000th based units. false = based on 1024; true = based on 1000

The return is a DisplayInUnits_return where it stores both the number (mostly between 0 to 999) and the string for the unit, like "KiB"
Maybe just a string would also do the job, but thats not relevant anymore.

Behaviour

we set up an index to 0 and then set up a bool to see if the decimal value is negative
if it is negtative we make the number positive.
then we set up the divider. Normally it is 1024, but if the 'kilo' bool is set, then we use 1000

Then we incement the index as long as the value is over 999, however if the index is bigger than the array of string, then we recude the index and break. then, if we did not break, we set the value to the value divided by the divider.

If the number started as a negtive number, we make it negative again. then we generate the DisplayInUnits_return and return it.
Note: in the code you will see Sizes[boolAsIndex(kilo), i], this is an 2d array holding the unit names, the first dimension is if we use KiB or KB, and we use a boolen as index
